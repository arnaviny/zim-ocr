Activity Diagram - ZIM OCR

flowchart TD A[Start] --> B[User uploads document] B --> C[File stored locally] C --> D[Run OCR on file] D --> E[Extract data from text] E --> F[Validate extracted data] F --> G[Save data to DB] G --> H[Update dashboard statistics] H --> I[Return JSON response] I --> J[End]

Explanation

This activity diagram outlines the overall process flow for the ZIM OCR system:

  • התחלה: תהליך העיבוד מתחיל.
  • משתמש מעלה מסמך: המשתמש בוחר ומעלה מסמך (PDF/תמונה) דרך ממשק ההעלאה.
  • אחסון הקובץ באופן מקומי: המערכת מאחסנת את הקובץ באופן זמני בסביבה המקומית לצורך עיבוד.
  • הרצת OCR על הקובץ: המערכת מפעילה מנוע OCR (כגון Tess4J) כדי להמיר את תוכן הקובץ לטקסט.
  • חילוץ נתונים מהטקסט: המערכת מפעילה אלגוריתם (כגון regex או NLP) לחילוץ המידע הרלוונטי מהטקסט שהתקבל מה-OCR.
  • אימות הנתונים החולצו: הנתונים שעברו חילוץ נבדקים לאימות ולהבטחת תקינותם.
  • שמירת הנתונים במסד הנתונים: הנתונים המאומתים נשמרים במסד הנתונים (PostgreSQL) במודל ה־Document.
  • עדכון נתוני הדשבורד: המערכת מעדכנת את הנתונים הסטטיסטיים והגרפיים בדשבורד בהתאם לשינויים במסד.
  • החזרת תגובת JSON: לבסוף, המערכת מחזירה תגובת JSON המכילה את כל המידע החולץ והמעובד ל־Frontend.
  • סיום: התהליך מסתיים.
